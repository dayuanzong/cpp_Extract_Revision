1k3d68 优化编程计划

目标
对 1k3d68.onnx 进行精度优化，2d106det.onnx 暂不优化，仅作为对比参考，不修改其流程。

当前逻辑差异总览（1k3d68 vs 2DFAN）
1. 模型类型
   - 1k3d68：InsightFace 关键点坐标回归模型（直接输出坐标）
   - 2DFAN：热力图回归模型（输出 68×64×64 heatmap）
2. 输入尺寸
   - 1k3d68：从模型输入形状读取，当前为 192×192
   - 2DFAN：固定 256×256
3. 裁剪方式
   - 1k3d68：以人脸框中心为基准，用固定比例系数裁剪后缩放到输入尺寸
   - 2DFAN：使用 scale=(w+h)/195.0 与 h=200.0*scale 的裁剪逻辑
4. 预处理归一化
   - 1k3d68：在 0..1 与 (x-127.5)/128 两种模式中自动选择
   - 2DFAN：固定 1/255 归一化
5. 通道与布局
   - 1k3d68：RGB，CHW
   - 2DFAN：BGR→RGB后写入 NHWC，再在输出解码中反向映射
6. 输出解码
   - 1k3d68：根据输出长度判断坐标维度（68×2 或 68×3）
   - 2DFAN：heatmap 最大值位置 + 次像素偏移
7. 后处理与鲁棒性
   - 1k3d68：无多采样，依赖单次裁剪的稳定性
   - 2DFAN：支持 5 点多采样求均值，稳定性更好
8. 与 2d106det 的关系
   - 1k3d68 与 2d106det 共用同一条 InsightFaceLandmark 代码路径
   - 2d106det 额外需要 106→68 映射用于对齐

优化方向（仅针对 1k3d68）
1. 裁剪比例独立化
   - 将 1k3d68 与 2d106det 的裁剪比例拆分，避免折中参数影响 1k3d68
2. 预处理模式锁定
   - 固定使用最匹配 1k3d68 的归一化模式，避免自动选择造成波动
3. 采样策略提升
   - 增加小范围多采样平均（仿照 2DFAN），提升稳定性
4. 输出一致性校验
   - 针对 1k3d68 的输出维度与范围加入自检，避免异常点影响对齐

实施步骤
1. 梳理 1k3d68 当前裁剪、预处理与输出路径
2. 增加 1k3d68 专用配置项（裁剪比例、归一化模式、采样开关）
3. 实现 1k3d68 多采样平均并与现有流程对齐
4. 添加 1k3d68 专用验证脚本与可视化对比
